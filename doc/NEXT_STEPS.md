## 🔄 Процесс управления изменениями

### Для каждой задачи:

1. **Создание ветки**
   ```bash
   git checkout develop
   git pull origin develop
   git checkout -b feature/task-name
   ```

2. **Разработка**
   - Маленькие, атомарные коммиты
   - Понятные сообщения коммитов
   - Регулярное тестирование

3. **Проверка качества**
   ```bash
   python scripts/code_quality_check.py
   pytest tests/
   ```

4. **Pull Request**
   - Заполнить шаблон PR
   - Запросить review
   - Исправить замечания

5. **Merge**
   - Только после одобрения
   - Squash commits если нужно
   - Удалить feature branch
