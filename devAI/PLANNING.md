### План: Историческое заполнение базы данных (Кардинально Обновлено)

**Цель:** Создать механизм для сбора, обработки и сохранения всех новостных статей с сайта `warandpeace.ru` за определенный период (начиная с 1 января 2025 года).

**Предпосылка:** Архив новостей на сайте доступен через форму поиска с параметрами `date_st` (дата начала) и `date_en` (дата окончания), а также с постраничной навигацией внутри результатов поиска. Пример URL: `https://www.warandpeace.ru/ru/archive/search/_/page=1/?text_header=&author=&topic=&date_st=01.01.2025&sselect=0&date_en=04.8.2025&archive_sort=`

---

### Этап 1: Исследование и доработка парсера (`src/parser.py`)

**Задача:** Научить парсер формировать запросы к архиву по датам и извлекать с них статьи вместе с точной датой публикации, учитывая внутреннюю пагинацию результатов поиска.

1.  **Анализ URL-структуры поиска:**
    *   **Действие:** Изучить предоставленный URL: `https://www.warandpeace.ru/ru/archive/search/_/page=1/?text_header=&author=&topic=&date_st=01.01.2025&sselect=0&date_en=04.8.2025&archive_sort=`
    *   **Цель:** Определить, как правильно формировать URL для запросов с разными датами и номерами страниц.
        *   `page=N`: номер страницы результатов поиска.
        *   `date_st=DD.MM.YYYY`: дата начала.
        *   `date_en=DD.MM.YYYY`: дата окончания.

2.  **Модификация `get_articles_from_page` (Переименование и адаптация):**
    *   **Действие:** Переименовать `get_articles_from_page` в `get_articles_from_archive_search(start_date_str, end_date_str, page=1)`.
    *   **Логика работы:**
        a. Функция будет принимать даты в строковом формате `DD.MM.YYYY` и номер страницы.
        b. Сформировать URL запроса, используя эти параметры.
        c. Выполнить `requests.get()` к этому URL.
        d. **Важно:** Сохранить текущую логику извлечения `title`, `link` и `published_at` (которую я уже начал реализовывать в предыдущем шаге).
        e. **Новое:** Добавить логику для определения общего количества страниц результатов поиска, чтобы `fetch_articles_in_date_range` могла корректно итерироваться.
    *   **Результат:** Функция будет возвращать список словарей статей, а также общее количество страниц для данного поискового запроса.

3.  **Создание новой функции `fetch_articles_in_date_range(start_date: datetime, end_date: datetime)`:**
    *   **Действие:** Я создам новую "умную" функцию, которая будет управлять процессом сбора статей.
    *   **Логика работы:**
        a. Преобразовать `start_date` и `end_date` в строковый формат `DD.MM.YYYY`.
        b. Начать цикл со страницы `page = 1`.
        c. В цикле вызывать `get_articles_from_archive_search(start_date_str, end_date_str, page)`.
        d. Собирать полученные статьи.
        e. Проверять, достигнута ли последняя страница результатов поиска.
        f. Увеличивать номер страницы и повторять, пока все страницы не будут обработаны.
    *   **Результат:** Функция вернет полный список статей, опубликованных в заданном диапазоне дат.

### Этап 2: Доработка базы данных (`src/database.py`)

**Задача:** Позволить базе данных сохранять статьи с указанной, а не только с текущей датой.

1.  **Модификация функции `add_article`:**
    *   **Действие:** Я изменю сигнатуру функции, добавив новый необязательный параметр: `add_article(url, title, summary, published_at_iso=None)`.
    *   **Логика работы:**
        a. Если `published_at_iso` передан, SQL-запрос `INSERT` будет использовать его для заполнения поля `published_at`.
        b. Если `published_at_iso` равен `None`, будет использоваться старая логика с `CURRENT_TIMESTAMP`.
    *   **Результат:** Мы сможем добавлять в базу как исторические, так и самые свежие статьи, не создавая конфликтов.

### Этап 3: Создание скрипта-заполнителя (`scripts/backfill.py`)

**Задача:** Написать одноразовый скрипт, который объединит все новые возможности и заполнит базу данных.

1.  **Создание файла `scripts/backfill.py`**.
2.  **Написание логики скрипта:**
    *   **Инициализация:** Импортировать все необходимые функции из `parser`, `summarizer` и `database`.
    *   **Сбор:** Вызвать `parser.fetch_articles_in_date_range(datetime(2025, 1, 1), datetime.now())` для получения полного списка статей с 1 января 2025 года по сегодняшний день.
    *   **Обработка:** Запустить цикл по полученному списку (от самых старых к самым новым).
    *   **Внутри цикла для каждой статьи:**
        a. Проверить, нет ли ее уже в БД (`is_article_posted`).
        b. Если нет:
            i. Получить полный текст (`get_article_text`).
            ii. Сделать краткое содержание (`summarize_text_local`).
            iii. Сохранить в базу с помощью доработанной `add_article`, передав точную дату публикации.
            iv. Вывести лог в консоль (`Добавлена статья за [дата]...`).
            v. Сделать паузу в 1-2 секунды, чтобы не создавать излишнюю нагрузку.

**Задача:** Написать скрипт для регулярного получения последних новостей с главной страницы и добавления их в базу данных.

1.  **Создание файла `scripts/fetch_latest_news.py`**.
2.  **Написание логики скрипта:**
    *   **Инициализация:** Импортировать необходимые функции из `src.parser` и `src.database`.
    *   **Сбор:** Вызвать `get_articles_from_page(1)` для получения статей только с первой страницы.
    *   **Обработка:** Запустить цикл по полученному списку статей.
    *   **Внутри цикла для каждой статьи:**
        a. Проверить, нет ли ее уже в БД (`is_article_posted`).
        b. Если нет:
            i. Получить полный текст (`get_article_text`).
            ii. Суммаризация отключена (как и в `backfill.py`).
            iii. Сохранить в базу с помощью `add_article`, передав точную дату публикации.
            iv. Вывести лог в консоль.
            v. Сделать небольшую паузу.
