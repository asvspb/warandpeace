# Уточненный план реализации функции «Дайджест за предыдущие сутки» (Выполнено)

**Общая цель:** Изменить логику генерации суточного дайджеста, чтобы она создавала отчет не за плавающие "последние 24 часа", а за строго определенный календарный день — **вчера** (с 00:00:00 по 23:59:59).

**Обоснование:** Это делает результат работы функции предсказуемым и консистентным. Дайджест за вчерашний день, запущенный сегодня в 9 утра, будет идентичен дайджесту, запущенному в 11 вечера.

---

### Этап 1: Модификация функции доступа к данным (`src/database.py`) (Выполнено)

-   [X] **Задача:** Изменить функцию `get_summaries_for_period`, чтобы она могла принимать точный диапазон дат, а не просто количество дней.
-   [X] **Действие:**
    -   [X] Переименовать функцию `get_summaries_for_period(days: int)` в `get_summaries_for_date_range(start_date: str, end_date: str)`.
    -   [X] Изменить SQL-запрос с `published_at >= datetime('now', ...)` на `published_at BETWEEN ? AND ?`.
    -   [X] Новая функция будет принимать две строки в формате ISO (`YYYY-MM-DD HH:MM:SS`) и использовать их в запросе.
-   **Обоснование:** Это сделает функцию более гибкой и позволит запрашивать данные за любой произвольный период, что является ключевым для новой логики.

### Этап 2: Обновление логики команды в боте (`src/bot.py`) (Выполнено)

-   [X] **Задача:** Реализовать вычисление точных границ "вчерашнего дня" и передать их в обновленную функцию базы данных.
-   [X] **Действие:**
    -   [X] В обработчике команды `/daily_digest` (функция `daily_digest_command`):
        1.  Получить текущую дату (`today = datetime.now()`).
        2.  Вычислить вчерашнюю дату (`yesterday = today - timedelta(days=1)`).
        3.  Определить начало вчерашнего дня: `start_of_yesterday = yesterday.replace(hour=0, minute=0, second=0, microsecond=0)`.
        4.  Определить конец вчерашнего дня: `end_of_yesterday = yesterday.replace(hour=23, minute=59, second=59, microsecond=999999)`.
        5.  Вызвать новую функцию из `database.py`: `get_summaries_for_date_range(start_of_yesterday.isoformat(), end_of_yesterday.isoformat())`.
-   **Обоснование:** Здесь будет реализована основная бизнес-логика, которая точно определяет, за какой период нужно собрать новости.
